# 视觉检测与 3D 位姿计算说明

本文档说明项目中两个核心视觉能力：
1. **调用厂商大模型服务进行目标检测或获取图像指定点坐标**
2. **将图像像素点转换到相机坐标系下的 3D 位姿**

---

## 一、调用厂商大模型服务进行目标检测

### 1.1 原理与流程

项目中通过 **OpenAI 兼容接口** 调用厂商的视觉大模型（如阿里云的 `qwen-vl-max` 或本地 `ollama` 部署的 `qwen2.5vl`）。
核心思路是：
- 将图像编码为 `base64` 格式；
- 通过 `system prompt` 限定模型角色和输出格式（必须返回纯 JSON 或坐标数组）；
- 通过 `user prompt` 指定要检测的目标或要返回的特定点位（如边界框、右上角点等）；
- 解析模型返回的文本，提取坐标。

### 1.2 代码示例

#### 方式 A：使用 `openai` 客户端调用远程服务（项目主要方式）

```python
from openai import OpenAI
import base64
import cv2
import json
import re

# 1. 初始化客户端（以阿里云 dashscope 为例）
client = OpenAI(
    api_key="your-api-key",
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1"
)

# 2. 读取/编码图像（支持 image_path 或 image_np）
image_path = "data/test_images/medicine.jpg"
with open(image_path, "rb") as f:
    base64_image = base64.b64encode(f.read()).decode("utf-8")

# 3. 构造 prompt，明确要求返回坐标
system_prompt = """你是一个专业的目标检测助手。任务是：
1. 分析图片中的物体；
2. 找到用户指定的目标；
3. 返回该物体的边界框左上角和右下角坐标；
4. 如果要获取特定点（如右上角），直接返回该点坐标。

规则：
- 坐标系：图片左上角为原点(0,0)，向右为x轴正方向，向下为y轴正方向
- 必须严格按纯文本JSON返回：{"x1": 200, "y1": 150, "x2": 500, "y2": 350}
- 未找到返回：{"x1": 0, "y1": 0, "x2": 0, "y2": 0}
- 不要输出任何多余文字"""

user_prompt = '请在图片中找到“氨咖黄敏胶囊”并返回边界框坐标。'

# 4. 调用模型
completion = client.chat.completions.create(
    model="qwen-vl-max-latest",
    messages=[
        {"role": "system", "content": [{"type": "text", "text": system_prompt}]},
        {
            "role": "user",
            "content": [
                {"type": "image_url", "image_url": {"url": f"data:image/jpeg;base64,{base64_image}"}},
                {"type": "text", "text": user_prompt}
            ]
        }
    ],
    temperature=0.1,
    max_tokens=100
)

result_text = completion.choices[0].message.content.strip()
print("模型返回:", result_text)

# 5. 解析坐标
json_match = re.search(r'\{[^}]*"x1"[^}]*"y1"[^}]*"x2"[^}]*"y2"[^}]*\}', result_text)
if json_match:
    coords = json.loads(json_match.group())
    x1, y1, x2, y2 = coords['x1'], coords['y1'], coords['x2'], coords['y2']
    print(f"检测框: ({x1}, {y1}) -> ({x2}, {y2})")
```

#### 方式 B：获取指定点（如物体的右上角点）

只需在 `user_prompt` 中明确要求即可：

```python
user_prompt = '请在图片中找到“红色药盒”，并返回它右上角的像素坐标，格式为 {"x": 300, "y": 200}'
```

### 1.3 关键注意事项

| 项 | 说明 |
|---|---|
| **图像格式** | 必须使用 `JPEG` 格式（`.jpg` 或 `.jpeg`），部分厂商 API 不支持 PNG |
| **坐标系约定** | 必须在 prompt 中反复声明：左上角为原点，向右为 `x`，向下为 `y` |
| **输出约束** | 大模型容易输出 markdown 代码块或解释性文字，必须在 system prompt 中严格禁止 |
| **容错处理** | 返回结果需用正则提取 JSON/数字，避免因模型额外输出导致解析失败 |

## 二、像素坐标转相机坐标系下的 3D 位姿

### 2.1 物理意义

相机成像的本质是一个**三维空间点到二维像素平面的投影**过程。反过来，如果我们知道：
- 一个像素点在图像上的坐标 `(u, v)`
- 该像素点对应的实际深度值 `Z`（即物体到相机成像平面的距离）

就可以利用相机的**内参**（焦距 `fx, fy` 和光心 `ppx, ppy`），通过**相似三角形原理**，将该像素点反投影回相机坐标系下的三维坐标 `(X, Y, Z)`。

### 2.2 数学推导（针孔相机模型反投影）

#### 正向投影公式

在相机坐标系下，一个 3D 点 `P_cam = (X, Y, Z)` 投影到像素平面的过程为：

```
u = fx * (X / Z) + ppx
v = fy * (Y / Z) + ppy
```

其中：
- `fx, fy`：相机在 x 和 y 方向上的焦距（单位：像素）
- `ppx, ppy`：光心（主点）坐标，即相机光轴与成像平面的交点
- `Z`：深度值，即点到相机光学中心的距离（单位需与最终 3D 坐标一致）

#### 反向推导（反投影）

将上述公式变形，求解 `X, Y`：

```
X = Z * (u - ppx) / fx
Y = Z * (v - ppy) / fy
Z = Z
```

这就是从像素坐标 `(u, v)` 和深度 `Z` 恢复相机坐标系下 3D 坐标的全部数学基础。

### 2.3 代码实现（项目中的核心逻辑）

```python
import numpy as np

# 输入
real_x, real_y = 320, 240       # 像素坐标 (u, v)
dis = depth_frame[real_y][real_x]  # 深度值，单位：毫米 (mm)

# 相机内参
color_intr = {
    "fx": 600.0,
    "fy": 600.0,
    "ppx": 320.0,
    "ppy": 240.0
}

# 反投影计算（结果单位为 mm）
x = dis * (real_x - color_intr["ppx"]) / color_intr["fx"]
y = dis * (real_y - color_intr["ppy"]) / color_intr["fy"]
z = dis

# 转换为米 (m)
x, y, z = x * 0.001, y * 0.001, z * 0.001

print(f"相机坐标系下 3D 位姿: X={x:.3f}m, Y={y:.3f}m, Z={z:.3f}m")
```

### 2.4 物理含义图解

```
          相机光心 (Optical Center)
                O
                |
                | Z (深度轴，光轴)
                |
    -----------成像平面-----------
    |                            |
    |      P (u, v)              |
    |        *                   |
    |        |                   |
    |        | dy = v - ppy     |
    |        |                   |
    |--------*-------------------|
    |        光心 (ppx, ppy)     |
    |                            |
    ------------------------------

在真实三维空间中：

          O (相机)
          |\
          | \
          |  \
          |   \  实际光线
          |    \
          |     * P_world (X, Y, Z)
          |    /
          |   /
          |  / Z = dis
          | /
          |/
```

- **`dx = u - ppx`**：像素点相对于光心的水平偏移量（像素）。
- **`fx`**：焦距（像素）。
- **`dx / fx`**：该偏移量对应的光线与光轴的夹角的正切值（`tan(θ)`）。
- **`Z * (dx / fx)`**：在深度为 `Z` 的平面上，该光线对应的实际水平距离 `X`。

`Y` 方向的推导完全对称。

### 2.5 从相机坐标系到机械臂基座坐标系（手眼标定）

相机坐标系下的 3D 点通常还需要转换到机械臂基座坐标系下才能用于抓取。项目中通过**手眼标定**得到：
- `rotation_matrix`：相机到机械臂末端的旋转矩阵
- `translation_vector`：相机到机械臂末端的平移向量

转换公式（齐次坐标变换）：

```python
import numpy as np
from scipy.spatial.transform import Rotation as R

# 相机坐标系下的点
obj_camera = np.array([x, y, z])

# 1. 构建相机到机械臂末端的齐次变换矩阵
T_cam2end = np.eye(4)
T_cam2end[:3, :3] = rotation_matrix
T_cam2end[:3, 3]  = translation_vector

# 2. 机械臂末端当前位姿 -> 齐次变换矩阵
end_pose = np.array([x1, y1, z1, rx, ry, rz])  # 位置 + 欧拉角
pos = end_pose[:3]
ori = R.from_euler("xyz", end_pose[3:], degrees=False).as_matrix()

T_base2end = np.eye(4)
T_base2end[:3, :3] = ori
T_base2end[:3, 3]  = pos

# 3. 计算物体在基座坐标系下的位姿
obj_homo = np.append(obj_camera, 1)
obj_base_homo = T_base2end @ T_cam2end @ obj_homo
obj_base = obj_base_homo[:3]
```

> **注意**：手眼标定矩阵的具体形式取决于标定方式（Eye-in-Hand / Eye-to-Hand），实际使用中需确认矩阵方向（是 `cam -> end` 还是 `end -> cam`），必要时取逆。

## 三、总结

| 步骤 | 作用 | 关键输入 | 关键输出 |
|---|---|---|---|
| **大模型视觉检测** | 获取目标在图像中的像素位置 | 原始图像 + Prompt | 像素坐标 `(u, v)` 或边界框 |
| **像素转 3D（反投影）** | 计算目标在相机坐标系下的位置 | 像素坐标、深度值、相机内参 | 相机坐标 `(X, Y, Z)` |
| **手眼坐标变换** | 将相机坐标转换到机械臂可执行的基座坐标 | 相机坐标、手眼标定矩阵、末端位姿 | 机械臂基座坐标 |

以上三个步骤串联起来，即完成了从 "看到" 到 "抓到" 的完整闭环。
